アイデア `$ARGUMENTS` の現在のフェーズを完了し、次のフェーズに進む。

## 手順

1. `ideas/$ARGUMENTS/README.md` を読み、現在のフェーズを特定する（🔄 のフェーズ）
   - 引数が空の場合は `ideas/` 配下の一覧を表示して選択してもらう

2. 現在のフェーズの完了チェック:
   - フェーズのドキュメントが存在し、内容が記載されていること
   - テンプレート（`_templates/`）の必須セクションが埋まっていること
   - 未解決の疑問点が残っている場合はユーザーに確認する

3. 判定ゲートの処理（該当する場合）:
   - **Phase 1→2**: 追求判定 — PoCの価値があるかユーザーに確認
   - **Phase 3→4**: Go/No-Go判定 — PoC結果を踏まえて進行可否をユーザーに確認
   - **Phase 4→5**: スコープ確認 — 要件の範囲が妥当かユーザーに確認
   - **Phase 6完了**: 着手判定 — 実装を開始するかユーザーに確認
   判定結果は該当フェーズのドキュメント末尾に記録する

4. `README.md` のステータスを更新する:
   - 現在のフェーズ: 🔄 → ✅ 完了 + 更新日
   - 次のフェーズ: ⬜ → 🔄 進行中 + 更新日
   - ドキュメントリンクを追加

5. 次のフェーズに応じた作業を案内する:
   - **Phase 2（PoC計画）**: `01-brainstorm.md` の内容を踏まえて PoC計画を策定。`_templates/02-poc-plan.md` を参照
   - **Phase 3（PoC実施）**: `02-poc-plan.md` の計画に沿って各PoC項目を実施するよう案内。各PoC項目ごとに `03-poc/<item>/` ディレクトリを作成
   - **Phase 4（要件定義）**: 全PoC結果を踏まえて要件を整理。`_templates/04-requirements.md` を参照
   - **Phase 5（タスク分解）**: 要件をタスクに分解。`_templates/05-tasks.md` を参照
   - **Phase 6（ロードマップ）**: タスクからロードマップを構築。`_templates/06-roadmap.md` を参照

6. Phase 6 が完了した場合、全体サマリーを生成してユーザーに提示する
